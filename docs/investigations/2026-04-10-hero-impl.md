## Component architecture

### HeroBalticMap layers (bottom to top)

| Layer | Type | Content |
|-------|------|---------|
| 0 | Raster base | Two `<image>` elements (dark/light PNG), opacity-switched by theme |
| 1 | Invisible cable paths | `<defs>` with 3 `<path>` elements as GSAP MotionPath routes |
| 2a | Visible cable traces | Subtle dashed stroke (0.25 opacity) colored by flow direction |
| 2b | Particles | 4 `<circle>` per cable, animated along paths via GSAP MotionPathPlugin |
| 2c | Flow labels | MW value + direction arrow at cable midpoints |
| 3 | Country revenue rings | `<circle>` with stroke-dasharray progress showing daily revenue share |
| 4 | Project dots | 9 `<circle>` elements at approximate substation locations with pulse animation |
| 5 | KPI tiles | HTML overlay (absolute-positioned), 4 tiles: live rate, annualised, S/D, fleet MW |

### Data fetching
Single `useEffect` fetching in parallel:
- `/s4/fleet` → S/D ratio, fleet MW by country, operational/pipeline totals
- `/revenue?dur=4h` → live_rate (€/MW/day, annualised), project_irr
- `/read` → DA capture (gross_4h)
- `/s8` → interconnector flows (NordBalt, LitPol, LT↔LV)

### Motion discipline
- `prefers-reduced-motion`: particles stop, dot pulse disabled, ring progress static
- GSAP via `useGSAP()` hook (React strict mode safe, auto-cleanup)
- Ring progress uses CSS `transition` (not JS animation)
- No looping decorative animation except particles (which represent live data flow)
